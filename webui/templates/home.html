The Distant Reader, a tool for reading stuff

Intended for anybody who reads, the Distant Reader takes text as input, does analysis against it, and outputs sets of structured data, affectionately called "study carrels". The student, research, or scholar can then use a study carrel to read the text both closely as well as at a distance.

What's here

  1. Library "catalog" - a simple & rudimentary list of curated study carrels
  2. Patrons / sign in - if you have created study carrels, then you can log in to see them here; if you would like to create study carrels, then request a username/passsword
  3. Interfaces for creating study carrels:
    • carrels with data you supply
      1. a file - quick & easy, but is really only useful for "large" files like entire books
      2. a Zip file - save files of just about any type in a folder/directory on your computer, compress (zip) the folder, and submit the results; great for those journal articles you downloaded for your research paper or the items in your class's reading list
      3. a URL - quick & easy, but the content may not be accesible by the Reader and the results may be overwhelmed with website navigation information; try Wikipedia pages, a blog, or a departmental website
      4. a list of URLs - like creating carrels from a single URL but more targeted and more scalable; consider creating a study carrel, and use the list of extracted URLs as the input for this service
    • carrels with data from external services
      • HathiTrust - millions of books in the public domain; search the 'Trust, add items to your "collection", download your or somebody else's collection file, and use it as input for this service
      • bioRxiv - a pre-print server for biology; search bioRxiv, mark items of interest, download the list of selected items as an "XML" file, and use the result as the input for this service
    • carrels with data from local indexes
      • Project Gutenberg - a local cache of the about 30,000 public domain documents from the venerable Project Gutenberg; submit a query, and create a carrel from the results; very easy and almost all queries will return something; great for "big" ideas (like love, war, or knowledge), "classic" authors (like Plato, Shakespeare, or Jane Austen), or types of literature (like Romances, Westerns, or Science Fiction)
      • CORD-19 - a cache of more than 100,000 scholarly and scientific journal articles on the topic of COVID-19; based on a data set by the same name, this index can be used to create study carrels based on topics (like infection, vaccines, or SARS) or journals titles (like Lancet, PLoS One, or Journal of Medical Virology)
  4. Project CORD - an "archived" website originally sponsored by the COVID-19 HPC Consortium for the purposes of indexing & providing access to CORD-19

About

Through the use of study carrels, the student, researcher or scholar can easily ask and answer questions such as but not limited to:

  • Who or what is mentioned in this corpus?
  • What do they do?
  • When does the action take place?
  • Where does the action take place?
  • How are they described?
  • What themes exist in a corpus and how are the manifested over time or between writers?

Study carrels are also useful tools for getting your mind around large corpora because they provide ways to sort, group, and prioritized their content.

The key to getting the most out of the Distant Reader is two-fold:

  1. articulating a question you would like to address, and
  2. identifying a corpus which may be used to answer the question

The questions can be as rudimentary as "What words are used most frequently in Moby Dick?" or as sublime as "How was love defined during the 19th Century?"

Study carrels can be created from but not limited to: websites, blogs, social media feeds, or files from your computer. The Reader can analyze just about any type of file including but not limited to: Portable Document Format (.pdf) files, Word (.doc and docx) documents, PowerPoint (.ppt and pptx) files, HTML (.htm and .html) files, plain text (.txt) files, XML (.xml) files, etc.

Software

From a computer system's point of view, the Distant Reader is really the amalgamation of differents sets of computer software tools used to create a (hopefully) useful and interesting service. Much of that software inclues, but is not limited to:

  • Airivata - used to submit and monitor large-scale applications
  • Archive::Zip - a Perl library to create Zip files
  • bootstrap - a library of Javascript and CSS functions
  • CGI - A Perl module for creating CGI scripts
  • csvstack - a set of command-line tool reading tabular data
  • DBI - a Perl library to do I/O against databases
  • File::Basename - a Perl library to process file names
  • File::Copy - a Perl library used to copy files
  • Gensim - a Python library for natural language processing
  • HTML::Entities - a Perl library for encoding data for the Web
  • jquery - a Javascript library for user-interface
  • JSON - a Perl library for reading JSON data
  • mpl_toolkits - a Python library for generating charts & graphs
  • networkx - a Javascript library for creating network diagrams
  • nltk - a Python library for natural language processing
  • OpenStack - a Linux toolkit for creating virtual machines
  • pandas - a Python library for processing matrix-based data
  • sklearn - a Python library for machine learning
  • Slurm - a Linux toolkit to support cluster computing
  • SpaCy - a Python library for natural language processing
  • SQLAlchemy - a Python library for interacting with databases
  • SQLite - a C library-based relational database
  • Textacy - a Python library for natural language processing
  • Tika - a Java archive for text pre-processing
  • WebService::Solr - a Perl library used to interact with a Solr index
  • Wget - a HTTP user-agent
  • wordcloud - a Python library for creating word clouds
  • XML::XPath - a Perl library for querying XML

All of the code to the Distant Reader is located in a number of different GitHub repositories. If you would like to contribute, then please see "Contributing to the Distant Reader". Software is never done, and if it were, then it would be called "hardware".
